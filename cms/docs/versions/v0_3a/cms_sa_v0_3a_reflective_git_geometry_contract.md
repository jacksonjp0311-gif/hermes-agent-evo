# CMS-SA v0.3a - Reflective Git Geometry Contract

## Purpose

Formalize Git history as reflective repository geometry.

## Core law

A commit is not only a change; it is a routed event in repository geometry.

## Secondary law

A repository learns when repeated routed events become constraints, validators, evidence surfaces, and future routing rules.

## Runtime object

ReflectiveGitNode includes:

- commit hash,
- parent links,
- tag,
- message,
- changed files,
- shell,
- meridian,
- sector,
- evidence reports,
- validator reports,
- lesson IDs,
- release truth context.

## README surface

v0.3a adds a dedicated Reflective Git Geometry Box to the root README.

## Non-claim lock

Reflective Git geometry improves repository orientation, routing, and evidence traceability. It does not prove code correctness, truth, AGI, consciousness, production readiness, security, external validation, or real-world correctness.
