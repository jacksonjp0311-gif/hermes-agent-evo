# CMS Version Registry

Every CMS version must be recorded as:

1. canonical document,
2. injection record,
3. version registry entry,
4. lineage ledger entry,
5. release seal,
6. roadmap next-anchor.

This prevents version drift where the code moves but the documents, injections,
README, RCC surfaces, and public narrative do not.

Non-claim lock:

```text
Version recording improves traceability.
It does not prove correctness, truth, AGI, consciousness, or external validation.
```

Registry artifact:

```text
outputs/version_registry/cms_version_registry.json
```
