# CMS-SA v0.3b1 - README and Mini README Reflective Feedback Alignment Lock

## Purpose

Ensure that root README sections, affected mini READMEs, validation reports, and version registry advance together after new durable surfaces are added.

## Boundary

This is not API implementation. It is a pre-API reflective feedback alignment lock.

## New Rule

If a version adds a durable surface, root README + mini README + validator + report + registry must advance together.

## New Validator

`scripts/validation/validate_surface_alignment_v0_3b1.py`

## Non-claim lock

Surface alignment improves repository navigation and reflective feedback alignment only.
