# CMS in 90 Seconds

Cybernetic Memory System is a local-first repository memory runtime.

It does not just store context. It closes a loop:

```text
observe -> measure -> drift -> feedback -> lifecycle -> plan -> validate -> promote -> release
```

Use this repo to test whether a project can:

1. observe its own state,
2. measure drift,
3. emit actionable feedback,
4. resolve feedback through lifecycle states,
5. generate a bounded next-version plan,
6. validate the plan,
7. promote only control-useful memory,
8. preserve correction history,
9. align README/RCC/public surfaces,
10. emit an evidence package.

Non-claim lock:

```text
Feedback is not truth.
Memory is not consciousness.
Surface alignment is not code correctness.
Validation is repository-bound.
```

## Thread Rehydration Protocol

Before proposing a new version in a fresh thread, scan:

1. Origin theory.
2. Software architecture.
3. Runtime state.

Core rule: theory tells why; architecture tells how; runtime tells now.

No fresh-thread versioning without Origin Scan, Architecture Scan, and Runtime State Scan.
