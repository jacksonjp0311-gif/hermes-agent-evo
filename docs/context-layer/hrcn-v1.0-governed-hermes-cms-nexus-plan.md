# HRCN v1.0 Governed Hermes-CMS Nexus Planning

Status: docs/context planning only.

## Primary Law

```text
The nexus may coordinate; it may not self-authorize.
```

## Purpose

HRCN v1.0 defines the governed nexus plan between Hermes, RCC, CMS, HRCN, evidence, rollback, and human authorization.

It is the architecture seal before the CMS read-only mirror/import authorization path.

```text
Hermes acts.
RCC orients.
CMS governs.
HRCN bridges.
Evidence bounds.
Human authorization unlocks dangerous transitions.
```

## Core Relation

```text
H = Hermes Agent Evo action runtime
R = RCC / RCC-N repository orientation layer
C = Cybernetic Memory System governance layer
N = HRCN bridge contract
E = Evidence package and benchmark harness
U = Human authorization boundary

Nexus = H o R o C o N o E o U
Operationally: R -> C -> E -> N -> H -> U
```

## CMS Read-Only Mirror Authorization Path

CMS is next, but v1.0 does not copy CMS.

The next recommended phase is:

```text
HRCN v1.0.1 - CMS Read-Only Mirror Import Authorization Path
```

A future CMS read-only mirror/import requires:

```text
CMS source repository or local source path
import method decision
source commit/hash or snapshot identifier
file manifest before copy
license/provenance note
secret scan before copy
read-only boundary statement
blocked-authority statement
rollback/removal plan
post-import validation commands
human authorization
```

Disallowed initial authorities:

```text
runtime integration
CMS write authority
memory write authority
API authority
repair apply authority
autonomous authority
security/production correctness claims
```

## Decision Rules

```text
No nexus plan can authorize itself.
No nexus plan can copy CMS in v1.0.
No CMS mirror can become authority merely by existing.
No evidence package can grant runtime or apply authority without explicit later gate and human authorization.
The next CMS step must be read-only mirror authorization, not runtime integration.
If CMS provenance, manifest, secret scan, rollback, validation, or authorization is missing, CMS import remains blocked.
```
